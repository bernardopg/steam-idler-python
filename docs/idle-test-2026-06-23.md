## Observed Issues

1. The terminal panel kept showing `Tempo idle` as `0 min` even after 10 and 20
   minute refreshes.
2. `run.sh`/Ctrl+C cleanup now stops native idling subprocesses, but the final
   session report still did not appear in the terminal log after interrupt.
3. There were pre-existing orphan/legacy `steam-utility idle` processes, which
   can make real tests ambiguous unless the target App IDs are cleaned first.
4. The refresh interval is hardcoded to 10 minutes; the 5-minute verification in
   this test had to be done externally.
5. immediate post-stop badge scraping may miss a card drop for a few seconds
   while Steam propagates account state.
6. Store API fallback logged `Error checking trading cards app 2321720: 'list'
   object has no attribute 'get'`, indicating parser handling for an unexpected
   appdetails payload shape is still missing.
7. Badge API returned no card-drop data for all 466 candidates, so the bot relied
   on authenticated web scraping for drop counts.
